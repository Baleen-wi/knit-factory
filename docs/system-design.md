# 系统设计

## 目标

为针织加工厂建立一个进货货款管理系统，覆盖每日进货记录、厂家对账、款号货款统计、时间段统计、付款登记和未付余额。

## 用户角色

| 角色 | 主要工作 |
| --- | --- |
| 录入员 | 录入每日进货记录、修改错误记录 |
| 财务 / 老板 | 查看厂家总货款、已付、未付、时间段汇总 |
| 对账人员 | 标记记录是否已对账，导出厂家对账资料 |

当前单机版没有登录。升级多人版时可加角色权限。

## 核心数据对象

### 进货记录 Delivery

| 字段 | 说明 |
| --- | --- |
| `id` | 唯一编号 |
| `date` | 进货日期 |
| `vendor` | 厂家 |
| `itemNo` | 款号 / 货号 |
| `color` | 颜色 |
| `unitPrice` | 单件价格 |
| `quantity` | 数量 |
| `reconcileStatus` | 对账状态：未对账 / 已对账 |
| `batchNo` | 单号或批次 |
| `note` | 备注 |

单笔总价不保存为核心字段，由 `unitPrice × quantity` 实时计算。

### 付款记录 Payment

| 字段 | 说明 |
| --- | --- |
| `id` | 唯一编号 |
| `date` | 付款日期 |
| `vendor` | 厂家 |
| `amount` | 付款金额 |
| `method` | 付款方式 |
| `note` | 备注 |

## 统计口径

| 指标 | 计算方式 |
| --- | --- |
| 所有货款 | 全部进货记录的 `unitPrice × quantity` 合计 |
| 时间段货款 | 日期筛选后的进货记录合计 |
| 厂家总货款 | 某厂家进货记录合计 |
| 款号总货款 | 某款号或款号 + 颜色的进货记录合计 |
| 已付货款 | 付款记录合计 |
| 未付余额 | 进货货款 − 已付货款 |
| 未对账笔数 | `reconcileStatus !== "已对账"` 的进货记录数量 |

## 当前技术形态

- `index.html`：页面结构
- `styles.css`：界面样式
- `app.js`：数据、计算、交互逻辑
- 浏览器 `localStorage`：本地保存数据

优点：无需安装、可离线使用、容易演示。

限制：只适合单台电脑和单个浏览器，不适合多人同时录入。

## 数据备份

当前系统支持：

- 完整 JSON 备份：包含进货记录、付款记录和版本信息
- 进货 CSV 导出：方便用 Excel 查看或打印
- 付款 CSV 导出
- JSON 导入：恢复完整备份
- CSV 导入：导入进货记录

## 后端升级路线

### 第一阶段：本地 SQLite 版

适合一台电脑长期使用，数据更安全。

建议技术：

- 前端：保留当前 HTML/CSS/JS 或改 React/Vue
- 后端：Node.js + SQLite
- 数据库表：deliveries、payments、vendors（可选）

### 第二阶段：局域网多人版

适合办公室多台电脑录入。

新增能力：

- 登录
- 操作员权限
- 多人同时录入
- 服务端备份
- 导出厂家对账单

### 第三阶段：云端版

适合异地查看、老板手机查看。

新增能力：

- 手机端适配
- 自动备份
- 审计日志
- 按月结算
- 付款提醒
- 厂家往来账单

## 建议下一批功能

1. 厂家资料管理：联系人、电话、地址、默认付款方式。
2. 款号资料管理：品类、尺码、默认单价、图片。
3. 厂家对账单打印：按时间段生成一张可打印账单。
4. 付款状态：未付、部分付款、已付清。
5. Excel `.xlsx` 直接导入，不需要先另存 CSV。
6. 操作日志：谁在什么时候改了哪条记录。
