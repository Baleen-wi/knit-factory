# 重构说明

## 本次重构定位

用户已说明 Excel 只是测试版。因此本次不再把 Excel 表结构当作最终需求，而是按“针织加工厂进货、对账、付款”的完整业务流程重构。

当前交付是本地单机网页版原型，重点验证：

- 数据对象是否合理
- 录入流程是否顺手
- 汇总口径是否清楚
- 后续接数据库时是否容易扩展

## 文件判断

| 文件 | 判断 |
| --- | --- |
| `交货记录汇总.xlsx` | 测试样例数据，可用来验证计算逻辑 |
| `hyper-v.txt` | Windows Hyper-V 包清单，与业务无关 |
| `Hyper.bat` | 启用 Hyper-V 的系统脚本，与业务无关 |
| `Π订单.docx` | 关系代数/数据库题目，与业务无关 |

## 新系统原则

1. 原始流水优先：所有统计从进货记录和付款记录实时计算。
2. 不维护手工汇总表：避免重复计账和汇总口径不一致。
3. 进货与付款分离：进货产生应付，付款冲减应付。
4. 对账状态独立：已对账不等于已付款。
5. 可以从单机版平滑升级到数据库版。

## 旧 Excel 样例中的口径问题

Excel 的「每日记录汇总」明细合计是：

- 总数量：242 件
- 总货款：¥2,625.00

旧「总汇总」显示：

- 总数量：404 件
- 总货款：¥4,264.00

差异来自“刘”厂家的数量和金额被重复统计了一次。新版系统避免维护多张手工汇总表，所有统计都从原始流水实时计算。
